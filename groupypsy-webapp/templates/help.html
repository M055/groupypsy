Help and Instructions

Application Usage

Upload the four required files on the Dashboard page. See details of the four files and examples of each below.
Once all files are uploaded, click the Process button to generate the output.
You can view the processed data or download it as a CSV file.

When you are done, you can clear the data from the server by clicking the Clear Data button.

Students File

The Students file contains information about each student and is primary indexed by Student_Num, a unique identifier for each student. Each row is one student, and the columns specify their details such as ID, first and last names, email address. There are N additional columns, where N is the number of Requirements. Each column title is the Requirement_Num for the corresponding Requirement, and values are 1 or 0 to indicate whether the student in that row meets (1) or does not meet (0) that requirement. The required columns are:

The required columns are:

  • Student_Num: A unique identifier for each student.
  • ID: The student ID number.
  • firstName: First name.
  • lastName: Last name.
  • emailAddress: The email address of the student.
  • Requirement_01...Requirement_N: The N Requirements, with the column names matching the Requirements table primary index, Requirement_Num.

Projects File

The Projects file lists all available projects, and is primary indexed by Project_Num. In addition to the name of the Supervisor, this table provides information on the Capacity of each project, as well as the project requirements. Each Requirement_N column has the title corresponding to the N primary indices of the Requirements table, Requirement_Num. For each row (project), the correspoding Requirement_ column indicates whether that equirement is (1) or is not (0) required for that project.

The required columns are:

  • Project_Num: A unique identifier for each project.
  • Supervisor: The name of the Supervisor.
  • Capacity: The maximum number of students permitted on this project.
  • Requirement_01...Requirement_N: Which of the Requirements are required for this project.

Example Projects File

Requirements File

The Requirements file lists the possible requirements across the various projects. This table is primary indexed by Requirement_Num. The Description column specifies details of that requirement.

The required columns are:

  • Requirement_Num: The unique identifier for the requirement.
  • Descriptiob: Human readable description of the requirement.

Example Requirements File

Choices File

The Choices file contains student preferences for projects, and has no primary index of its own. Each row is a student, with the Student_Num correspoding to the values of the Student file. Each column is a project, with the Project_Num correspoding to the values of the Project file. In each row (i.e., for each student), the various columns have values from 1...N, corresponding to the N projects, with the most desirable project with a value 1 and the least desired with a value N.

The required columns are:

  • Student_Num: Student identifiers, corresponding to the Student table.
  • Project_01...Project_N: Project identifiers, corresponding to the Projects table.
